# Pattern Library

La mémoire produit du système. Chaque brief est un distillat de recherche
(Mobbin, web, expérience) validé par l'usage — c'est l'actif qui rend ce
système plus fort à chaque projet.

Règles :
- Un fichier par pattern (onboarding-kyc.md, paywall.md, empty-states.md…).
- Format strict : _template.md. Une page max.
- TOUJOURS consulté avant toute recherche externe (étape 2 de /feature).
- Un brief contredit par l'expérience se met à jour, ne se supprime pas
  (garder la trace du "on rejette").
- Préfixer les patterns spécifiques à un domaine (`fintech-kyc.md`, `health-consent.md`) ; les patterns universels restent sans préfixe (`empty-states.md`).
