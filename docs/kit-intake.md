# Journal d'intake /retro

Registre des rapports `/retro` (produits côté projet) effectivement portés et
appliqués dans le kit. Il ferme la couture manuelle du /retro : un rapport
distillé mais jamais appliqué = apprentissage perdu, sans ce journal pour le
rendre détectable.

**Règle.** Chaque session kit qui applique un rapport /retro ajoute une ligne
ci-dessous, dans le même commit que les amendements. Un rapport reçu mais non
encore appliqué se note en `⏳ en attente` jusqu'à son application.

| Date rapport | Projet d'origine | Amendements | Statut | Commit |
|---|---|---|---|---|
| 2026-06-17 | kit (self) | 3 (retro kit-self, proposé/validé, résolution captures) | ✅ appliqué | _(ce commit)_ |
| _(exemple)_ 2026-06-13 | wc2026 | 3 (anti-slop, exemplaire, /critique) | ✅ appliqué | `aa34117` |
